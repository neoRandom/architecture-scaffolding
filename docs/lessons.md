## Future refactors and improvements

Currently, the app is good enough to help me define and manage projects' architecture, but it's still lacking features and having an imprecise architecture model.

Some refactors and improvements include:

- Lean the whole building design more into interfaces and contracts

- Split 'Service' into Domain Services and Application Services

- Add a toggle system to mark when a node is done being coded.

- Add rules and hard constrains

- Remove the step thing and make what now is the second step as the main interface, with a title-only representation like how the first step is now.

- Integrate it with the workflow and the data file saved within the project root.

- Support for multiple projects

The whole project needs to be rebuild from scratch.
